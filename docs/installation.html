Installation

To install the current release of desc you can choose:

  • PyPI
    Directly install the package from PyPI:
$ pip install desc

If you do not have sudo rights (you get a Permission denied error):

$ pip install --user desc
  • Github
    Download the package from the Github and install it locally:
git clone https://github.com/Yafei611/desc
cd desc
pip install .
  • Anaconda

If you do not have a working Python3.5 or Python3.6 installation, consider installing Anaconda , (see Installing Anaconda ).After installing Anaconda, you can creat your environment, for example DESC (you can change other name.)

conda create -n DESC python=3.5.3
# activate your environment 
source activate DESC 
git clone https://github.com/Yafei611/desc
cd desc
python3 setup.py build
python3 setup.py install
# now you can check whether `desc` installed sucessfully!


Run a quick desc test

Now, let us run a quick test to see if the desc is ready:

Firstly, enter Python’s Interpreter from your console:

$ python

Then, in an Python interpeter session, typing:

>>> import desc
>>> desc.run_desc_test()

The run_desc_test() function enables you to perform a quick desc analysis on a small pre-processed data. If you see something like:

“The desc has been trained successfully!”
…
“The run time is xxx.xxxx”

then you are ready to go.

Troubleshooting

  • For linux user: If you see */ld: cannot find -l*** , which means it cannot find lib file, where l*** stands for link lib file lib***.so. The first thing you need install the cooresponding lib file, for example if you saw */ld: cannot find -lxml2, you can try 
    #For ubuntu/debian user you can try
sudo apt-get install python3-dev
sudo apt-get install libxml2-dev
#For ubuntu/debian user you can try
sudo yum install libxml2-devel
sudo yum install libxml2-devel

    Then try to install desc again. If issues still arise, you should try the following (but with Permission), we take libxml2 as example

    cd /usr/lib64
ls -al|grep "xml2"
sudo ln -s libxml2.so.2 libxml2.so
ls -al|grep "xml2"

  • For windows user You can click python-igraph if there are some issues for python-igraph (DESC depends on python-igraph). Note that the automatic compilation of the C core when running pip install python-igraph will not work on Windows! So the preferred way to install the Python interface on Windows is to download one of the unofficial installers built by Christoph Gohlke. (Thanks Christoph!). Download it according to your python version and windows, but because python-igraph depend on pycairo, you had better download pycairo firstly.

  • Take python3.6 win64 as example,after you download pycairo and python-igraph, you can run following commands. 
    pip install pycairo-1.18.0-cp36-cp36m-win_amd64.whl
pip install python_igraph-0.7.1.post6-cp36-cp36m-win_amd64.whl
  • If you have some install issues about pydot,
    • For linux users, you should can run pip3 install pydot graphviz.
    • For windows user, you should download Graphviz from Graphviz and you need to set the PATH variable yourself. After you finished the following setting, you should close the cmd window and restart cmd window. 
        #here I downloaded `graphviz` in C:\Users\dell\Desktop
  import os
  graphviz_setup='C:\Users\dell\Desktop\graphviz-2.38\release\bin'
  os.environ["PATH"] += os.pathsep + graphviz_setup
